## What this unit is about

This unit introduces one of the most important ideas in beginner hardware programming:

**input → code → output**

That means:
- the micro:bit receives an action
- the program detects it
- the device responds

In this workshop, participants will use **buttons** as input, the **LED display** as output, and a **variable** to keep track of information.

## What we will learn

By the end of this unit, participants should be able to:
- understand the difference between **input** and **output**
- use the **MakeCode blocks editor** more confidently
- create programs with **button A**, **button B**, and **A+B**
- display icons, text, and patterns on the LED screen
- use a simple **variable** such as a counter
- build a small interactive micro:bit project

## Main concepts in this unit

### 1. Input
Input is what the device receives.

Examples:
- button A pressed
- button B pressed
- buttons A+B pressed

### 2. Output
Output is what the device shows or does.

Examples:
- display an icon
- scroll text
- light up LEDs
- play a sound

### 3. Variables
A variable stores information that can change.

In this workshop, participants may use a variable such as:
- `count`
- `score`
- `clicks`

## Workshop plan

### Part 1 — Explore button events
Participants begin with a simple activity using button events.

Example:
- button A shows a heart
- button B shows a smiley face
- A+B plays a short melody or shows a symbol

This helps everyone understand how the micro:bit can react to actions.

### Part 2 — Add visual feedback
After the first button actions work, participants can improve the project by adding:
- scrolling text
- a custom LED pattern
- an icon or symbol
- a short sound effect

This makes the project more personal and more fun.

### Part 3 — Use a variable
Next, participants add a simple variable.

Example:
- each time button A is pressed, increase `count` by 1
- show the updated value on the screen

This introduces a very important programming idea in a beginner-friendly way.

## Suggested mini project

### Button Counter
Participants build a small interactive project where:
- button A increases a counter
- button B decreases the counter or resets it
- A+B shows the current number

This is a strong Unit 2 project because it combines:
- events
- LED output
- variables
- simple project logic
